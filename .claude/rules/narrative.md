# Narrative Document Rules

- All narrative docs use Markdown
- Lore entries must be cross-referenced with a systems-index entry
- Define canon levels: CORE (immutable), ESTABLISHED (stable), FLEXIBLE (changeable)
- Character dialogue maximum 120 characters per line (iOS screen width)
- All dialogue must be localization-ready — no culturally specific idioms without alternatives
- World rules must be documented and consistently applied
- Mystery elements must have answers documented (even if never revealed)
- Faction relationships must use consistent logic documented in a relationship matrix
